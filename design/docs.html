The product

A loom reads top to bottom as prose, but its code is written in whatever order explains it best. The product is what the pieces become once assembled: real source, composed from the sections the anchors name, with the narration left behind.

01What the product is

Loom is not a templating engine but a composition system, and code is its product. Within a chapter a section draws in another by a ::[…] anchor; the product pass resolves those anchors into one composed whole, and a {Tangle} section binds it to the file path that tangle writes.

The prose never appears in the product. Tangle discards it, keeping only the code and the comments an author meant to ship. What a person reads and what a machine runs are two layers of one file, and the product is only the second of them.

02Composing by anchor

A section names another with an anchor rather than pasting its code inline. The order on the page is the reader's; the order in the file is the compiler's. Below, the module is assembled last from three sections defined earlier, each under the prose that explains it.

03Why keep the layers apart

Knuth's premise was that a program is a work of literature. Prose explains to a person what we want a computer to do; code tells the computer to do it. Loom keeps the two distinct so neither has to carry the other's weight — the prose can wander for clarity, and the product stays exactly the source the language expects.

Instead of imagining that our main task is to instruct a computer what to do, let us concentrate rather on explaining to human beings what we want a computer to do. — Donald E. Knuth, Literate Programming (1984)

04Reviewing the product

Because the product runs, it can be reviewed as it runs. Loom Notes draws an overlay over the page, so a reader points at what they see and leaves a note where it belongs; a coding agent reads those notes and works through them. The next page shows every one of the overlay's components at rest.
